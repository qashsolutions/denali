# UI/UX

## Principles

- Minimal interface â€” just a chat box
- No forms, no dropdowns, no medical jargon
- Mobile-first (Medicare patients often on phones/tablets)
- Accessibility (large text option, high contrast)
- Greeting personalization ("Evening, Venkata")
- Smart suggestions appear below input (tappable)
- Response formatting:
  - Short paragraphs
  - Bold for emphasis sparingly
  - Printable checklist at the end
- Loading states ("Checking Medicare coverage...")
- Feedback mechanism (simple thumbs up/down)

---

## Design System

### Theme Toggle

Users can switch between **Dark** and **Light** modes:
- Toggle location: Header (sun/moon icon) + Account Settings
- Default: Follow system preference
- Persisted to user preferences (localStorage for free users, database for paid)

- Light mode provides better readability in bright environments (important for elderly users)

---

## Implementation Notes

### CSS Variables Approach

```css
:root {
  /* Light theme (default follows system) */
  --bg-primary: #f8fafc;
  --bg-secondary: #ffffff;
  --text-primary: #0f172a;
  --text-secondary: #475569;
  --border-color: #e2e8f0;
  --accent: #2563eb;
}

[data-theme="dark"] {
  --bg-primary: #0f172a;
  --bg-secondary: #1e293b;
  --text-primary: #ffffff;
  --text-secondary: #94a3b8;
  --border-color: #334155;
  --accent: #3b82f6;
}

@media (prefers-color-scheme: dark) {
  :root:not([data-theme="light"]) {
    /* Apply dark theme variables */
  }
}
```

### Theme Toggle Logic

```javascript
// Check system preference
const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches;

// Check stored preference
const storedTheme = localStorage.getItem('theme');

// Apply theme
const theme = storedTheme || (prefersDark ? 'dark' : 'light');
document.documentElement.setAttribute('data-theme', theme);
```
